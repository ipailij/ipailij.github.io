kilobyte(kB, 10^3^), Megabyte(MB, 10^6^), Gigabyte(GB, 10^9^), Terabyte(TB, 10^12^), Petabyte(PB, 10^15^), Exabyte(EB, 10^18^), Zettabyte(ZB, 10^21^), Yottabyte(YB, 10^24^), Ronnabyte(RB, 10^27^), Quettabyte(QB, 10^30^)

kibibyte(kiB, 2^10^), Mebibyte(MiB, 2^20^), Gibibyte(GiB, 2^30^), Tebibyte(TiB, 2^40^), Pebibyte(PiB, 2^50^), Exbibyte(EiB, 2^60^), Zebibyte(ZiB, 2^70^), Yobibyte(YiB, 2^80^), Robibyte(RiB, 2^90^), Quebibyte(QiB, 2^100^)

Three Vs

  • Volume

  • Variety

    New shapes of data have emerged:

    • Trees

      XML, JSON, Parquet, Avro.

    • Unstructured

      Data accumulated in a raw form like text, pictures, audio, video.

    • Cubes

      Popular in business analytics.

    • Graphs

      Useful when the use case is focused on the efficient traversal of data, which is equivalent to joins in tables, like neo4j, Oracle PGX.

  • Velocity

    • Capacity: how much data can we store per unit of volume?
    • Throughput: how many bytes can we read per unit of time?
    • Latency: how much time do we need to wait until the bytes start arriving?

​ Discrepancy between capacity, throughput and latency -> parallelization(discrepancy between capacity and throughput) & batch processing (discrepancy between throughput and latency)
